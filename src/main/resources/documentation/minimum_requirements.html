Minimum Requirements

smAIrtRx - OR Implementation Guide

1. System Environment

  • Java Runtime: Java 21 or higher is required.
  • Build Tool: Apache Maven (for compilation and dependency management).
  • Hardware: Minimum 4GB RAM (8GB recommended for large datasets and complex optimizations).

2. Data Volume Requirements

To ensure statistical significance and meaningful optimization results, the input dataset must meet the following minimum volume:

Minimum Requirement: The CSV file must contain at least 100 rows of valid surgical data.

The application includes an automated ValidationTestFull that will fail if the dataset does not meet this threshold, preventing the generation of unreliable reports.

3. Data Preparation Workflow

The system enforces a strict three-stage process in the Add Data section to maintain mathematical accuracy:

  1. Selection: Choose a local CSV file. Path and filename are displayed.
  2. Validation: Run the VALIDATE_ONLY scan. The system checks against the Data Integrity Rules (see Section 4). If the file is missing headers or required columns, a simplified error report is shown and optimization is disabled.
  3. Optimization: Activation of the analysis engine requires a successful validation pass or the generation of a Cleaned Data version. Upon success, the interface automatically navigates to the Dashboard view.

4. CSV File Specification

The program expects a CSV file with specific headers. While header names can be mapped in app.properties, the following data points are mandatory:

Requirement Default CSV Header Description
Unique Identifier Case_ID A unique alphanumeric ID for each surgical case.
Procedure Date Date The date of the surgery (formatted per app.properties).
Facility Name Site The name of the hospital or clinic.
Operating Room OR_Room The specific room name or number.
Surgeon Surgeon_Name The name of the performing surgeon.
Procedure Type Procedure_Type The description or code of the surgery.
Start Timestamp (Labor) Anesthesia_Start_Time The time anesthesia began (used for Labor Yield).
End Timestamp (Labor) Anesthesia_End_Time The time anesthesia concluded (used for Labor Yield).
Room In (Physical) Patient_In_Room_Time The time the patient entered the OR (Primary driver for Room Consolidation).
Room Out (Physical) Patient_Out_Room_Time The time the patient exited the OR (Primary driver for Room Consolidation).

5. Data Integrity Rules

The following data conditions will cause a row to be ignored by the optimizer:

  • Rows Missing Required Data: Any row with a blank value in a mandatory column.
  • Cross-Midnight Durations: Cases where the End Time occurs before the Start Time (based on Physical Occupancy).
  • Procedures under min duration: Cases where the calculated physical duration is below the min.procedure.duration.minutes threshold.
  • Procedures over max duration: Cases where the duration exceeds the configured max.procedure.duration.hours threshold.
  • Malformed Dates/Times: Data that does not match the primary or fallback parsing patterns.
  • Duplicate Case IDs: Duplicate records with the same Case_ID.

6. Configuration (app.properties)

Ensure the following keys are correctly configured before execution:

  • csv.path: Absolute or relative path to the data file.
  • bin.capacity.minutes: Typically 480 for an 8-hour shift.
  • start.time.pad.minutes: Default 10 minutes.
  • end.time.pad.minutes: Default 10 minutes.
  • max.procedure.duration.hours: Default 12 hours.
  • min.procedure.duration.minutes: Default 15 minutes.
  • padding.procedure.[NAME]: Optional procedure-specific padding (e.g., 5,5).
  • date.format: Matches your CSV date style (e.g., M/d/yyyy).
  • time.format: Matches your CSV time style (e.g., hh:mm a).
